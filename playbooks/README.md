# Playbooks — Reusable Frameworks & Methodologies

Field-tested methodology library for the sales/presales motion this plugin supports —
research, POV-building, and presentation construction. Each playbook captures a pattern
that worked once and is likely to work again; customize to the client's context, don't
apply verbatim.

## Available Playbooks

| Playbook | Status | Use When |
|---|---|---|
| [Customer Research Methodology](customer-research-methodology.md) | v1.0 | Running the 5-step deep research cycle on a new client or prospect |
| [Sales Pattern Intelligence](sales-pattern-intelligence.md) | starter template | Feed this via `tvt-sales-distill` from your own meetings — ships empty of content by design (see the file itself for why) |
| [Sales Skills SDK](sales-skills-sdk.md) | v0.1 | Reusable "Lego block" reference for composing a sales motion from smaller, proven moves |
| [Compelling AI Sales PPTX](compelling-ai-sales-pptx.md) | — | Structuring the CONTENT of an AI solution presentation — what goes on each slide and why |
| [Slide Deck Visual DNA](slide-deck-visual-dna.md) | — | Structuring the VISUAL layer of a deck — hierarchy, density, layout storytelling |
| [Consolidated-View Explainer Formula](consolidated-view-explainer-formula.md) | — | Building a multi-part consolidated-view explainer document (Venn diagrams, swimlanes, From→To→Why tables) |

## How to Use

1. **Before a client engagement**: check if a playbook applies
2. **During engagement**: customize the playbook to the client's context
3. **After engagement**: note what worked, what didn't — playbooks are living documents, version them (v1.0, v1.1, ...)

## Contribution Rules

- Keep them practical: frameworks, templates, checklists — not theory
- Anonymize client/colleague specifics before adding a new pattern here — this directory
  ships to other installers, not just your own workspace
