RESEARCH — customer / vertical / competition intel
g-intel-pipeline — the master orchestrator; runs the full research package end to end, resumable.
"run the pipeline on [Company]"
g-intel-deep — first deep-dive research cycle on a new prospect.
"research [Company]" / "deep dive on [client]"
g-intel-customer — the 5-step deep cycle — financials, platform, insider knowledge, market, case studies. Reserve for top-ranked accounts.
"customer intel on [Company]"
g-intel-dossier — narrative account story + relationship map (org chart).
fired automatically inside the pipeline
g-intel-fanout — scans research/transcripts for named competitors/vendors, researches each.
"research all the competitors mentioned"
g-intel-qbr — quarterly business review prep — internal signal + industry + peer + tech-frontier triangulation.
"prep QBR for [Account]"
g-intel-factcheck — verifies every claim in an intel file against external sources — mandatory before a POV.
"fact check [file/directory]"
g-intel-dispatch — parallel research fan-out across many prospects at once (cloud agents).
"research all 10 credit-union targets"
g-intel-flywheel — extracts patterns that repeat across 2+ clients into reusable playbooks — the cross-chain multiplier (§1a).
"what patterns are we seeing across clients"
PRODUCT / AI-OPPORTUNITY
g-pm-jtbd — scores a client's under-served outcomes (Ulwick ODI) — finds where AI actually pays off.
"what AI opportunities does [Client] have"
g-pm-grow — grows scored opportunities into a roadmap + narrative (RICE×Kano + Thompson + MDI).
"what should we build and pitch to [Client]"
THIS SALES ENGINE — spec 006's new components
tvt-sales-engine — the router — any sales intent, fires the right job. Working — routes to tvt-sales-prospect/tvt-sales-pov/g-intel-*.
"who should I focus on this week" / "status" / "pipeline"
tvt-sales-prospect — confidence-scored shortlist under a capacity ceiling, protected floor per motion. Working + tested.
python score.py --rank --opportunities-file ... --capacity 8 --floor "hunt=3,expand=2"
tvt-sales-pov — synthesizes a testable Point of View / Business Value Hypothesis. Stage 1 (curated provocations) + job-5 verification gate: working + tested. Stage 2 (flywheel synthesis) gated behind Stage 1 field signal.
"build a POV for [target]"
kpi.py / eval_backtest.py — portfolio scorecard + the method of eval — Brier score, calibration, pattern reuse rate, case-study yield.
python kpi.py --opportunities-file ... --quota-hunt ...

PRESENTATION

"build a deck for [Client]"

"make the case for [architecture/idea]"

fired as part of deck-building
GOVERNANCE (fires automatically, §7 — not manually invoked)

"discovery call tomorrow — help me structure an approach"
